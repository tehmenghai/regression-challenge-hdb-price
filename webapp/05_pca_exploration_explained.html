Overview

This notebook applies Principal Component Analysis to 150,634 training rows using all 55 numerical columns, after median imputation and StandardScaler normalisation. The goal is diagnostic: understand how many truly independent dimensions the data has, and which features are the primary drivers of variation.

The answer is sobering for anyone hoping to compress the feature set: you need 22 principal components to explain 80% of variance, and 33 for 95%. This is not a low-dimensional dataset. The scree plot shows a gentle, sustained decline rather than a sharp elbow — every feature is carrying some unique information.

On the positive side, the 2D scatter confirms that PC1 correlates with resale price (higher-priced flats cluster at higher PC1 values), and the loading analysis pinpoints exactly which features matter most. These findings directly inform feature engineering choices in subsequent modelling notebooks.

Section 1 — Select Numerical Features

PCA operates only on numbers. After dropping id, resale_price (the target), and Tranc_YearMonth, 55 numerical columns remain. These span:

  • Flat attributes: floor area, storey, age, max floors, lease commencement
  • Block composition: unit-mix counts (1-room to executive, sold and rental)
  • Proximity: distances and counts for MRT, malls, hawker centres, primary and secondary schools
  • Coordinates: flat, MRT, school, bus stop latitudes and longitudes
DROP = ['id', 'resale_price', 'Tranc_YearMonth']
num_cols = [c for c in train.select_dtypes(include='number').columns
            if c not in DROP]
# → 55 numerical features across 150,634 rows

Section 2 — Impute & Scale

PCA requires complete, comparable data. Two preprocessing steps handle this:

  1. Median imputation — fills NaNs without distorting distributions (robust to outliers vs. mean)
  2. StandardScaler — centres and scales each feature to mean=0, std=1, so no large-magnitude feature (e.g., latitude ~1.3 vs. mall count ~5) artificially dominates the principal components
imputer  = SimpleImputer(strategy='median')
X_imp    = imputer.fit_transform(X_raw)
scaler   = StandardScaler()
X_scaled = scaler.fit_transform(X_imp)
# → 150,634 rows × 55 features, zero-mean unit-variance

High-dimensionality warning: The first three PCs together explain only 35.6% of variance. This rules out aggressive PCA compression for regression — a 10-component reduction would discard over 60% of the information.

The scree plot shows a shallow, extended decline with no sharp elbow. This pattern indicates many features each contribute a modest, non-redundant amount of variance — exactly the kind of dataset where gradient boosting trees thrive, since they can use all 55 features simultaneously without caring about scale or orthogonality.

Section 5 — Feature Loadings: What Drives PC1 & PC2?

Each loading is the weight a feature gets in the linear combination defining that PC. Positive = moves with the PC; negative = moves against it.

Limitations

  • PCA only uses the 55 numerical columns — categorical features (town, flat_type, flat_model) are excluded and may carry additional structure
  • PCA maximises variance, not price predictability; a feature can be low-loading in PCA yet highly predictive in regression
  • PCA features are not used as model inputs in subsequent notebooks — this analysis is purely diagnostic

Key Takeaways

  • High-dimensional data confirmed: 22 PCs needed for 80% variance — no shortcut dimensionality reduction.
  • Floor area dominates PC1, directly aligning with EDA correlation findings and justifying size-based interaction features.
  • Geography forms a distinct second axis (PC2), validating investment in lat/lon features, MRT distance engineering, and town encoding.
  • Price gradient is visible in 2D PCA space but noisy — full-feature gradient boosting trees will significantly outperform PCA-compressed models.
  • The feature set is information-rich with low redundancy — each of the 55 numerical features contributes something unique, supporting the wide feature set used in v6–v8 model notebooks.
