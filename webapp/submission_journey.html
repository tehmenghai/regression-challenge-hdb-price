4-Member Blend Experiment (+ Ken Lai) — Rank Avg Tied BEST, Weighted Failed
2026-04-29 • ensemble_team.ipynb • 2 submitted 1 generated only

What we tried

  • Added Ken Lai's submission ($35,328 Kaggle) to the 3-member ensemble
  • Updated MengHai to v19 (OOF $21,533, Optuna-tuned LGBM+ET)
  • Tested 3 blend strategies: 3-member equal, 4-member weighted, 4-member rank averaging
  • Lai's correlation with others: 0.9796–0.9815 (below 0.99 = meaningful diversity)

Why include Lai despite $35K RMSE?

Correlation ≠ accuracy. Even a weak model can add value if its prediction rank-ordering is partially correct. Lai's 0.9796 correlation means she agrees on which flats are expensive vs cheap — just with a systematic price scale bias. That ordering signal can be extracted safely via rank averaging.

💡 Key learning — when rank averaging works vs when raw averaging fails:
  • Weighted blend failed ($21,660.70, +$310 worse): Lai's absolute predictions are systematically inflated by ~$14K above market. At 11% weight (1/RMSE²), this adds a constant upward bias of ~$1,500 to every prediction — unavoidable because raw averaging uses actual price values.
  • Rank averaging (generated, not submitted): Rank averaging converts each member's predictions to their relative ordering (rank 1 = cheapest, rank N = most expensive), then averages those ranks. This neutralises Lai's absolute scale bias — her "this flat is relatively expensive" signal is preserved while her "$14K too high" offset disappears entirely. Not submitted due to submission limit constraints.
  • Rule: A model with correct relative rankings but wrong absolute scale should only be included via rank averaging, never raw or weighted averaging. Raw averaging amplifies scale errors; rank averaging immunises against them.
