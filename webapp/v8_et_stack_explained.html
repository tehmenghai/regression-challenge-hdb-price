The v8 idea in one sentence: We kept the v7 Ridge-on-OOF stacking framework but added a 3rd base model — Extra Trees Regressor — giving the meta-model a third independent "opinion" to blend, squeezing out another $101 in RMSE.

1. What Was Wrong With v7?

v7 used two base models — tuned XGBoost and tuned LightGBM — with a Ridge meta-model trained on their out-of-fold (OOF) predictions. It scored $21,906 on Kaggle.

Both XGB and LGBM are gradient boosting algorithms. They're related: they both build trees sequentially where each tree tries to correct the previous one's residuals. Because they're similar in nature, their predictions are correlated. The meta-model doesn't get much new signal by seeing both — they tell a similar story.

Analogy: Ask two friends who went to the same school the same question — you'll get similar answers. Now ask someone with a completely different background, and you get a genuinely independent perspective. The more diverse the opinions, the better your final consensus.

That's exactly what Extra Trees brings to v8.

2. What Is Extra Trees (Extra Randomized Trees)?

Extra Trees (ExtraTreesRegressor) is a bagging ensemble of decision trees — but with one extra twist compared to Random Forest: the split thresholds are chosen completely randomly.

Property Random Forest Extra Trees
Tree building Bootstrap samples (with replacement) Full training set (no bootstrap by default)
Feature selection at each split Random subset of features Same — random subset
Split threshold Best threshold found by search Completely random threshold
Variance Lower (bootstrap helps) Lower still (randomness averages out)
Bias Moderate Slightly higher (random splits aren't optimal)
Training speed Moderate Fast (no split search)

Why Does Random Splitting Help?

When XGB or LGBM predicts a price, they do it through a careful sequential process — each tree focuses on the hardest remaining errors. Extra Trees predicts through many shallow, randomly-split trees and averages them.

The result: ET makes different kinds of errors than XGB/LGBM. Its mistakes don't correlate with theirs. When the Ridge meta-model sees three OOF columns — XGB, LGBM, ET — the ET column provides genuinely new information.

Why ET alone is weaker ($23,123 val RMSE vs XGB's $21,769): Random splits are by definition not optimal. ET is intentionally "less smart" per tree — its strength is diversity and speed, not individual accuracy. It's a contributor, not a leader.

3. The v8 Architecture

Training data (5-fold CV)
    │     ├── → XGBoost (tuned)     → xgb_oof[n_train]     ├── → LightGBM (tuned)   → lgbm_oof[n_train]     └── → Extra Trees (tuned) → et_oof[n_train]                                   │                                   ▼             Ridge meta-model trained on [xgb_oof, lgbm_oof, et_oof]                                   │                                   ▼             Test predictions = Ridge( xgb_test, lgbm_test, et_test )                                   │                                   ▼                            Kaggle RMSE: $21,804.67

Every base model generates predictions on the training data it never saw (out-of-fold). This is critical — if we used in-fold predictions, the meta-model would learn from memorised training data, not genuine generalisation.

4. Out-of-Fold (OOF) Predictions — Why and How

OOF is the key that makes stacking honest. Here's the 5-fold process for one base model (same for all three):

for fold 1 of 5:
    train XGB on folds 2,3,4,5  →  predict on fold 1  →  xgb_oof[fold_1_rows]
for fold 2 of 5:
    train XGB on folds 1,3,4,5  →  predict on fold 2  →  xgb_oof[fold_2_rows]
... (repeat for folds 3, 4, 5)

# After 5 iterations: xgb_oof covers ALL training rows
# Each row was predicted by a model that never trained on it

We do this independently for XGB, LGBM, and ET. The result is a matrix of shape (n_train, 3):

Row xgb_oof lgbm_oof et_oof actual price
0 480,200 479,850 483,400 481,000
1 320,100 319,600 324,200 321,500
… … … … …

Ridge regression is then trained on this 3-column matrix. It learns the optimal weighted combination of the three base model predictions.

6. Ridge Meta-Model — How It Weights the Three Models

Ridge regression (linear regression with L2 regularisation) learns coefficients for each base model's OOF predictions. The coefficients tell us how much the meta-model trusts each base model:

XGBoost
~0.55
LightGBM
~0.40
Extra Trees
~0.10

The coefficients sum to approximately 1.0. XGB and LGBM dominate — as expected, since they're individually more accurate. ET gets a small but positive weight, meaning it contributes real signal even though it's weaker on its own.

Why does Ridge (not plain linear regression) work here? XGB and LGBM predictions are highly correlated. In plain OLS, correlated predictors cause unstable coefficients (one could go very large positive, another very large negative). Ridge's L2 penalty keeps coefficients stable and appropriately sized.

8. Complete v8 Pipeline Code

from sklearn.ensemble import ExtraTreesRegressor
from sklearn.linear_model import Ridge
from sklearn.model_selection import KFold
import numpy as np

kf = KFold(n_splits=5, shuffle=True, random_state=42)

# Storage for OOF predictions
xgb_oof  = np.zeros(len(X_train))
lgbm_oof = np.zeros(len(X_train))
et_oof   = np.zeros(len(X_train))

xgb_test_preds  = []
lgbm_test_preds = []
et_test_preds   = []

for fold, (train_idx, val_idx) in enumerate(kf.split(X_train)):
    X_tr, X_val = X_train[train_idx], X_train[val_idx]
    y_tr, y_val = y_train[train_idx], y_train[val_idx]

    # --- XGBoost ---
    xgb_model.fit(X_tr, y_tr)
    xgb_oof[val_idx] = xgb_model.predict(X_val)
    xgb_test_preds.append(xgb_model.predict(X_test))

    # --- LightGBM ---
    lgbm_model.fit(X_tr, y_tr)
    lgbm_oof[val_idx] = lgbm_model.predict(X_val)
    lgbm_test_preds.append(lgbm_model.predict(X_test))

    # --- Extra Trees ---
    et_model.fit(X_tr, y_tr)
    et_oof[val_idx] = et_model.predict(X_val)
    et_test_preds.append(et_model.predict(X_test))

# Average test predictions across 5 folds
xgb_test  = np.mean(xgb_test_preds,  axis=0)
lgbm_test = np.mean(lgbm_test_preds, axis=0)
et_test   = np.mean(et_test_preds,   axis=0)

# Stack OOF predictions and train Ridge meta-model
meta_train = np.column_stack([xgb_oof, lgbm_oof, et_oof])
meta_test  = np.column_stack([xgb_test, lgbm_test, et_test])

ridge = Ridge(alpha=1.0)
ridge.fit(meta_train, y_train)

final_predictions = ridge.predict(meta_test)

9. Why the Stack OOF RMSE ($21,708) Is Lower Than Kaggle ($21,804)

This is a healthy, expected gap. The OOF RMSE is measured on training data — rows the base models weren't trained on within that fold, but still rows from the same distribution as the full training set.

Kaggle data is a completely held-out test set, drawn from a possibly slightly different time period or distribution. A gap of ~$96 means our stack generalises very well — there's no sign of overfitting.

Version CV/OOF RMSE Kaggle RMSE Gap
v6 (fixed blend) $21,818 $22,124 $306
v7 (2-model stack) $21,842 $21,906 $64
v8 (3-model stack) $21,708 $21,805 $97

The gap collapsed dramatically from v6 to v7 (when we switched from fixed blending to OOF stacking), and stayed low in v8. This tells us the stacking framework is correctly estimating generalisation performance.

Glossary

TermPlain-English meaning
OOF (Out-of-Fold)Predictions made on the held-out validation fold — never seen during that model's training. Honest predictions.
Base modelFirst-level model that predicts the target directly (XGB, LGBM, ET in v8)
Meta-modelSecond-level model that learns to combine base model predictions (Ridge in v8)
StackingUsing one model's predictions as input features for another model
BaggingBuild many independent trees on random subsets; average results. Reduces variance.
BoostingBuild trees sequentially; each one fixes previous errors. Reduces bias.
Ridge (L2)Linear regression with a penalty on large coefficients — prevents instability when inputs are correlated
RMSERoot Mean Squared Error — average prediction error in the same units as the target (SGD)
