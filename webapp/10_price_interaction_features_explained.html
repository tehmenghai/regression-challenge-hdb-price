Key insight: Encoding the interaction of flat type × postal sector gives the model a direct price signal for each (location, flat type) pair — capturing up to a $595K gap that a sector-only encoding misses entirely.

Overview

Previous versions encoded geographic price signals at the postal-sector level, but gave every flat type in a sector the same baseline price. A 5-room in Queenstown sector 1425 sells for ~$845K while a 2-room in the same sector sells for ~$250K. The single postal_sector_median_price cannot distinguish these — it reports the same number for both rows.

This notebook introduces three new interaction OOF target encodings that pair resale price with two categorical dimensions simultaneously. All are computed out-of-fold to prevent leakage, and sparse groups fall back to a parent-level encoding rather than the global median.

A secondary fix corrects hdb_age, which in the raw data was computed using a hardcoded 2021 reference year, making it inconsistent with remaining_lease (which uses Tranc_Year). After the fix, hdb_age + remaining_lease = 99 for every row — as it should be for a 99-year HDB lease.

2. Feature Engineering

hdb_age Fix

The raw dataset computed hdb_age = 2021 - lease_commence_date (a fixed snapshot). This contradicts remaining_lease = 99 - (Tranc_Year - lease_commence_date), which uses the actual transaction year. For a flat sold in 2012: hdb_age + remaining_lease = 34 + 74 = 108 — nine years off. The fix:

# Before (raw data): hdb_age = 2021 - lease_commence_date  ← wrong
# After (fix):
df['hdb_age'] = df['Tranc_Year'] - df['lease_commence_date']
# Now: hdb_age + remaining_lease = 99 always ✓

Composite Key Columns

Three string keys are built by concatenating existing columns. They are used only as groupby keys for encoding — they are dropped from the model's feature matrix X afterwards:

df['ft_sector_key'] = df['flat_type'] + '_' + df['postal_sector']   # e.g. "4 ROOM_1424"
df['yr_sector_key'] = df['Tranc_Year'].astype(str) + '_' + df['postal_sector']  # e.g. "2018_1424"
df['yr_ft_key']     = df['Tranc_Year'].astype(str) + '_' + df['flat_type']      # e.g. "2018_4 ROOM"

3. Per-Fold OOF Target Encoding

Out-of-fold encoding ensures each row's encoded value is computed from the other four folds only — the row never contributes to its own price signal. This prevents data leakage while still producing an encoded value for every training row.

Why OOF matters: If you compute town_median_price using all training data and then train on it, the model is partly memorising the target. OOF breaks this — each row's encoding comes from data it wasn't trained on.

The two sparse interaction features fall back to postal_sector_median_price — not the global median — because that's a much tighter estimate for any rare group.

Sparsity risk: yr_sector_key has ~5,133 groups with 44.5% having fewer than 10 rows. Fold 2's outlier result ($22,460 vs ~$21,700 in other folds) shows this instability in action.

Key Takeaways

  • Interaction encodings address a genuine gap. ft_sector_median_price gives the model a direct price signal for each (flat type, location) combination — a 5-room and 2-room in the same sector differ by up to $595K, and the model can now learn this directly.
  • Sparsity is the main risk. yr_sector_median_price has 44.5% of groups with fewer than 10 rows. Fold 2's outlier ($22,460) shows the instability this causes. If the Kaggle score regresses, dropping this feature is the first experiment to try.
  • The hdb_age fix removes contradictory noise. Raw hdb_age used 2021 as reference, causing hdb_age + remaining_lease to sum to 108 for 2012 transactions instead of 99. Both features now consistently reflect the flat's age at time of sale.
  • Ridge OOF is optimistic. The meta-train RMSE ($21,716) is always lower than the true Kaggle score. Based on the v9 gap (+$128), the estimated Kaggle RMSE is ~$21,844 — which could go either way relative to v10's $21,755.
  • Stacking adds ~$228 over the best individual model. The diversity between XGB, LGBM, and ET is preserved — all three Ridge coefficients are positive and meaningful.
