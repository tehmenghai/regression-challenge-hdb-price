3. CV vs Kaggle Gap Trend

The gap between CV/OOF RMSE and Kaggle RMSE measures how well the model generalises. A smaller gap = the CV score is a reliable predictor of Kaggle performance.

Rule of thumb: Gap < $300 = well-calibrated. Gap < $100 = excellent. Gaps above $500 usually signal target leakage or overfitting to training data.

v14b
Lease Depreciation + Storey Interaction Features
2026-04-28 • 14b_clean_features.ipynb • team sub only

What we tried

  • log_remaining_lease: log1p transform — linearises depreciation curve
  • lease_below_60: binary flag for <60yr remaining (financing cliff)
  • lease_x_below60: remaining_lease × lease_below_60 interaction
  • flat_type_rank: ordinal size rank (studio → EA)
  • storey_x_flattype: mid_storey × flat_type_rank interaction

Why

HDB prices don't depreciate linearly with lease. The <60-year threshold is a market discontinuity — bank financing restrictions kick in. Storey premium differs by flat type: high floors matter more for large flats.

v16
4-Model Stack — CatBoost as 4th Base + 100% Data Retrain
2026-04-28 • 16_catboost_stack.ipynb • 🏆 best solo OOF

What we tried

  • Added CatBoostRegressor as 4th base (symmetric/oblivious trees)
  • CAT params: iterations=1000, depth=7, lr=0.08, l2=1, col=0.7
  • Ridge meta on 4 OOF predictions; alpha=0.001
  • Also generated 100% data retrain (sub_v16_fulldata.csv)

Ridge weights: XGB 0.343 + LGBM 0.367 + ET 0.120 + CAT 0.173

Why

CatBoost uses symmetric (oblivious) trees — it splits the same feature at every level, making structurally different errors than XGB/LGBM gradient boosting. Ridge exploits this diversity. 100% retrain uses all 150K rows (vs 80%/fold) for test predictions.
