MengHai's individual pipeline (v16)

Why 4 base models?

HDB prices are driven by both location (town, street, postal sector) and flat characteristics (model, storey, lease). No single algorithm captures all interactions equally.

  • XGBoost + LightGBM — different tree-split patterns; LightGBM is faster on high-cardinality data
  • ExtraTrees — random splits add bagging diversity; errors partially uncorrelated with boosters
  • CatBoost — symmetric trees learn different patterns; handles categoricals natively

Why Ridge meta-learner?

Ridge learns the optimal blend weight for each base model across folds — adapting to which model is stronger for different price segments.

  • Fixed blending (e.g. 50/50) is arbitrary; Ridge finds data-driven weights
  • L2 regularisation prevents over-relying on one model when OOF signals are noisy
  • Interpretable: weights tell us which model the market trusts most

⚙️ Section 2 — Data Processing & Feature Engineering

2. Feature Engineering

Why OOF target encoding over one-hot or label encoding?

MethodHowProblem
One-hot 26 towns × 20 flat models = hundreds of sparse columns Trees waste splits on low-variance columns
Label encoding Assign integers to categories Implies false ordering ("BEDOK=3 > ANG MO KIO=1" means nothing)
OOF median Group median resale price, computed on out-of-fold rows only Injects real market signal; no target leakage
Did we validate this? Partially — but not a clean head-to-head comparison. In v8 → v9, we switched from ordinal postal encoding (9,125 raw block levels treated as integers) to OOF postal_sector_median_price and Kaggle improved $21,805 → $21,755 (−$50). This shows OOF beats label/ordinal encoding for postal — but the granularity also changed (block-level → sector-level), so it is not a pure A/B test. We did not run a controlled ablation replacing OOF with one-hot for town or flat_model. The broader claim is supported by theory and Kaggle convention, not a direct experiment.
What we did validate (v8 → v9): Switching from ordinal postal encoding to OOF postal-sector median gave −$50 Kaggle RMSE. Adding postal_sector_median_price (598 groups) captures micro-location price gradients that town_median_price (26 groups) misses entirely.

Why per-fold re-encoding inside the stacking loop?

If we computed OOF encodings once globally and reused across folds, the fold-validation rows' prices would leak into their own feature values. We recompute the group median inside each fold using only fold-train rows. This adds ~30s per run but eliminates leakage — the OOF RMSE stays a true holdout estimate.

Key interaction features

storey × flat_type

A high floor in a 1-ROOM flat carries a larger relative premium than in a 5-ROOM. Multiplying storey rank by flat type rank captures this non-linearity.

lease_below_60 flag

CPF financing rules change when remaining lease falls below 60 years, creating a structural price break. A binary flag lets the model learn this discontinuity explicitly.

lease × floor area

Older + larger flats depreciate differently than older + smaller. The interaction captures this combined non-linearity that neither feature alone can express.

Why Optuna Bayesian HPO over grid/random search?

  • Grid search — with 8+ params per model, 8&sup8; = 16M combinations; infeasible
  • Random search — explores uniformly; wastes trials in low-quality regions
  • Optuna TPE — builds a probabilistic model of which parameter regions are promising; focuses trials there. 100 trials ≈ near-optimal with a fraction of compute

Why team ensemble as the final step?

Each member trained a completely different model (different features, algorithm, HPO). Their test-set errors are partially uncorrelated.

Averaging two predictions where errors partially cancel reduces RMSE even if one model is weaker — the same mathematical reason 4-model stacking beats any single model.

The CatBoost overfitting lesson (v20)

What happened: CatBoost Optuna was tuned on a single 80/20 random split. Best trial (depth=10, 1994 iters) minimised loss on one specific 20% holdout.

Inside our 5-fold CV, those params overfit — CatBoost OOF RMSE $21,780, but Ridge gave it 36.6% weight and Kaggle came back $21,827 (+$362 gap vs OOF $21,465).

Fix: Always tune inside the same CV strategy as training (5-fold CV in the Optuna objective function).

Why 5-fold KFold (not TimeSeriesSplit)?

Train data spans 9 years; test set is drawn from the same distribution. TimeSeriesSplit would penalise early folds unfairly by training on only a small slice of history.

The OOF-Kaggle alignment across v7–v16 (gaps under $200) validated this choice empirically.

Ridge weight monitoring as an overfitting detector

The CatBoost weight jumping from 17% to 36.6% was the warning sign — before we even saw the Kaggle result, we knew something unusual had happened with v20's CatBoost params.
🏆 Section 5 — Wins, Failures & What’s Next

5. Wins, Failures & What’s Next

✅ What Worked

🥇

OOF target encoding

postal_sector + flat_model + town×flat_type + year. Captured the market's own price signal without leakage. Largest structural improvement in the pipeline.

🥈

CatBoost as 4th stacker (v16)

Symmetric tree structure learns different patterns than XGB/LGBM asymmetric trees. Diversity alone added ~$250 OOF improvement. Zero OOF-Kaggle gap at v16.

🥉

3-member team ensemble

$300 improvement over best individual with no new model training — purely from error cancellation across 3 different modelling approaches.

✅

MLflow + CONFIG-driven pipeline

Eliminated ~400 lines of boilerplate per experiment. One CONFIG dict drives the entire pipeline; MLflow logs everything automatically.

❌ What Didn’t Work

🚫

H3 geospatial hexagon features

Engineered sub-km location features, but postal_sector OOF already captured the location signal more cleanly. Redundant and slower to compute.

🚫

CatBoost Optuna on 80/20 split (v20)

Seemed best in isolation (OOF $21,465) but overfit on Kaggle ($21,827, +$362 gap). Lesson: always tune inside the same CV as training.

🚫

6-member team ensemble

More members ≠ better blend. Weaker models (Hueyling $23k, Lai $22.5k) diluted stronger ones. 6-member equal blend ($21,754) worse than 3-member focused ($21,257).

🚫

Comparable sales features

Rolling 6-month comp median by town+flat_model — the idea was right but OOF encodings already saturated that signal. Adding both added noise (+$225 OOF).

🔬 What’s Next — Given No Time Constraint

🔧

Fix CatBoost HPO

Retune with 5-fold CV inside Optuna to close the $362 OOF-Kaggle gap. Should make v20-class models competitive individually.

⚖️

OOF grid search for ensemble weights

Instead of 1/RMSE² heuristic, minimise ensemble OOF RMSE directly over blend weights. More principled and data-driven.

📋

(town, flat_model, year) triple OOF encoding

Captures "how have 4-ROOM Model A flats in Bedok moved over time?" — more granular than current separate pairs.

🧠

Neural network as 5th stacker

TabNet or simple MLP — entirely different inductive bias from tree methods; potentially orthogonal errors that further diversify the stack.

👥

Every member improves their individual model

Each 1% improvement in any individual model feeds through to the team ensemble for free — compounding gains with no ensemble overhead.
